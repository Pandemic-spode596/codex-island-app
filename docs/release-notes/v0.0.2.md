# Codex Island v0.0.2

[English](#english) | [中文](#中文)

## English

Compared with `v0.0.1`, `v0.0.2` builds on the remote-host support that was already present and focuses on making both remote and local sessions easier to inspect, switch, and resume without ending up in a stale or confusing chat state.

### New Features

- Remote chat headers now show the active model, reasoning effort, service tier, and remaining context percentage.
- Local chat headers now show the active model and remaining context percentage parsed from transcript events.
- Remote chat now supports an explicit `/new` flow for starting a fresh thread even when another thread already exists under the same SSH target and working directory.
- Remote `/resume` now reads from the raw thread directory, making it much easier to reopen older same-directory threads.
- The app now shows each remote session's working directory and filters remote-session candidates by default working directory more consistently.
- Remote diagnostics are now off by default and only write `remote-app-server.jsonl` after you enable `Remote Debug Logs` from the menu.
- Interactive decision cards improve local multi-question approval and input flows.

### Bug Fixes

- Fixed remote slash-command handling for current Codex wire shapes and older app-server responses.
- Fixed remote open-session hydration, error handling, and rebinding so switching after `/new` no longer snaps back to a newer same-directory thread.
- Fixed the first local `Open Session` flow that could open an empty placeholder chat before transcript history was ready.
- Fixed delayed local transcript loading and local chat rebinding, reducing blank or stale chat state on first load.
- Fixed local session identity collisions across windows and working directories, including Ghostty matching issues.
- Fixed several notch and settings interactions, including hit-test bounds, hover-open behavior, reachable menu rows, and quit fallback.

### Notes

- Release build version: `0.0.2 (2)`
- This release continues to support both local hooks-based sessions and SSH remote hosts running `codex app-server`.

### Install

1. Download `CodexIsland-0.0.2-macOS-unsigned.zip`.
2. Unzip it and move `Codex Island.app` to `Applications`.
3. Launch the app manually.

### Release Notes

- This build is unsigned and not notarized because this environment does not have Developer ID signing, notarization credentials, or Sparkle release keys configured.
- macOS may show security warnings on first launch. Use the standard Gatekeeper override flow if you trust the build.

## 中文

相对于 `v0.0.1`，`v0.0.2` 是在已经存在的远端主机能力基础上，继续补强本地与远端会话的可见性、切换稳定性和恢复体验，减少进入聊天后出现过期、空白或错绑状态的情况。

### 新特性

- 远端聊天头部现在会显示当前模型、reasoning effort、service tier，以及剩余上下文百分比。
- 本地聊天头部现在会根据 transcript 事件显示当前模型和剩余上下文百分比。
- 远端聊天新增显式 `/new` 流程，即使同一个 SSH target 和工作目录下已有线程，也可以稳定新开一个 thread。
- 远端 `/resume` 现在基于 raw thread 目录取候选，重新打开同目录旧 thread 更稳定。
- 应用现在会展示远端会话的工作目录，并且按默认工作目录筛选远端会话候选更一致。
- 远端诊断日志现在默认关闭，只有在菜单里手动开启 `Remote Debug Logs` 后才会写入 `remote-app-server.jsonl`。
- 新增交互式 decision cards，改进本地多问题审批与输入流程。

### Bug 修复

- 修复远端 slash command 对当前 Codex wire shape 和旧版 app-server 响应的兼容性问题。
- 修复远端已有会话的打开、hydration、错误处理和 `/new` 之后的 rebinding，切到新 thread 后不再被同目录较新的 thread 自动顶回。
- 修复本地首次 `Open Session` 可能先出现空白占位聊天窗口的问题。
- 修复本地 transcript 延迟到达时的聊天 rebinding 与历史加载问题，减少首次进入时的空白或过期状态。
- 修复本地会话在不同窗口和工作目录下的 identity 冲突，包括 Ghostty 匹配异常。
- 修复 notch 和设置菜单的多处交互细节，包括命中区域、hover-open、菜单底部可达性和退出兜底行为。

### 说明

- 本次发布版本号：`0.0.2 (2)`
- 本版本继续同时支持本地 hooks 会话和通过 SSH 连接的远端 `codex app-server`。

### 安装

1. 下载 `CodexIsland-0.0.2-macOS-unsigned.zip`。
2. 解压后把 `Codex Island.app` 移到 `Applications`。
3. 手动启动应用。

- macOS 首次启动时可能会给出安全提示；如果你信任这份构建，可以按标准 Gatekeeper 放行流程处理。
