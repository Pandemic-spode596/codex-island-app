# Codex Island v0.0.4

[English](#english) | [中文](#中文)

## English

Compared with `v0.0.3`, `v0.0.4` is a hardening release. It focuses on making remote app-server sessions more diagnosable and resilient, surfacing hook and watcher failures instead of dropping them silently, and cleaning up the parser, state, and chat UI internals so future changes are less risky. It also tightens the project's quality and release tooling.

### Highlights

- Remote host support is more robust: the app now keeps more app-server state aligned across reconnects, preserves richer failure context, and adds better diagnostics around transcript fallback, transport flow, and remote thread selection.
- Remote host setup is easier: the `SSH Target` field now suggests aliases and resolved hosts discovered from local SSH config, while still allowing raw `user@host` input.
- Hook installation, socket handling, transcript watchers, and JSONL readers now report explicit failure reasons instead of relying on silent fallbacks, which makes local session issues much easier to debug.
- The local/remote chat, transcript parser, session store, and tool result rendering code were split into smaller focused components, reducing risk for future changes without changing the overall product model.
- Repository quality gates are stronger: staged Swift checks, CI quality/build coverage, heuristic audit calibration, and unsigned release fallback packaging are now all part of the documented release path.

### Notes

- Release build version: `0.0.4 (4)`
- Since `v0.0.3`, the repository has added `62` commits, mostly around remote session stability, hook/watch diagnostics, internal refactors, test coverage, and release automation.

### Install

1. Download `CodexIsland-0.0.4-macOS-unsigned.zip`.
2. Unzip it and move `Codex Island.app` to `Applications`.
3. Launch the app manually.

### Release Notes

- This build is expected to publish as an unsigned, non-notarized zip if Developer ID signing, notarization credentials, or Sparkle release keys are not configured in the current environment.
- macOS may show security warnings on first launch. Use the standard Gatekeeper override flow if you trust the build.

## 中文

相对于 `v0.0.3`，`v0.0.4` 更偏向一次补强型发布。重点是让远端 app-server 会话更稳、更容易诊断，把 hooks 和 watcher 的失败路径从静默吞掉改成显式暴露，同时把 parser、状态层和聊天 UI 内部拆分得更清晰，降低后续改动风险；此外也继续补齐了仓库的质量与发布工具链。

### 重点变更

- 远端主机链路更稳了：应用现在会在重连、fallback、transport 流转和 thread 选择上保留更多 app-server 状态与诊断信息，远端问题更容易定位。
- 远端主机配置更顺手：`SSH Target` 现在会基于本机 SSH 配置提示 alias 和解析后的 host，但依然保留直接输入 `user@host` 的能力。
- hooks 安装、socket 处理、transcript watcher 和 JSONL 读取现在都会记录明确失败原因，不再主要依赖静默 fallback，本地会话异常更容易查。
- 本地/远端聊天、transcript parser、session store 和 tool result 渲染代码被拆成更聚焦的组件，在不改变产品形态的前提下，降低后续维护和迭代风险。
- 仓库质量门禁继续补强：新增 staged Swift 检查、CI 中的质量与构建覆盖、启发式审计校准，以及无签名环境下的 unsigned release fallback 打包流程。

### 说明

- 本次发布版本号：`0.0.4 (4)`
- 从 `v0.0.3` 到现在，仓库累计新增 `62` 个提交，主要集中在远端会话稳定性、hooks/watch 诊断、内部重构、测试覆盖和发布自动化。

### 安装

1. 下载 `CodexIsland-0.0.4-macOS-unsigned.zip`。
2. 解压后把 `Codex Island.app` 移到 `Applications`。
3. 手动启动应用。

### 发布说明

- 如果当前环境没有配置 Developer ID 签名、公证凭据或 Sparkle 发布密钥，本次构建预计会以未签名、未公证的 zip 形式发布。
- macOS 首次启动时可能会给出安全提示；如果你信任这份构建，可以按标准 Gatekeeper 放行流程处理。
