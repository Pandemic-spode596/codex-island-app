# Codex Island v0.0.3

[English](#english) | [中文](#中文)

## English

Compared with `v0.0.2`, `v0.0.3` moves much more of the local-session experience onto the live local `codex app-server` path. Local chats can now open fresh threads, resume older ones, surface plan follow-up choices more reliably, and keep history and pending interactions aligned with the real local thread instead of falling back to stale transcript-only state. This release also tightens notch hit-testing and startup stability.

### New Features

- Local chat now runs much more like the remote flow: it can talk to a hidden local `codex app-server`, prefer live thread history, and keep local pending interactions in sync with the actual thread state.
- Local chat now supports `/new` and `/resume`, including reopening older local threads that share the same working directory.
- Local slash/config panels now cover `/plan`, `/model`, and `/permissions` for local app-server threads.
- Collapsed notch status now groups sessions into running, waiting, and idle buckets with count badges, so mixed local and remote activity is easier to scan at a glance.

### Bug Fixes

- Fixed several local plan-completion edge cases so follow-up choices remain visible after `request_user_input`, `turn_complete`, or `task_complete`, including transcripts that only emit a final `<proposed_plan>` block.
- Fixed local inline replies for app-server-backed interactions, so plan and choice-style prompts can be answered directly in the UI when the local thread is writable.
- Fixed the collapsed-notch hover hit area and prevented prompt taps inside the notch from leaking a `mouseUp` event through to the underlying fullscreen terminal window.
- Fixed local session recovery and identity handling by preferring visible local app-server threads, supporting synthetic sessions when needed, and reducing blank or mismatched history states.
- Fixed debug startup crashes caused by `SIGPIPE` during early local app-server or Sparkle-related writes.

### Notes

- Release build version: `0.0.3 (3)`
- Since `v0.0.2`, the repository has added `22` commits touching local app-server flows, plan interactions, notch behavior, startup stability, and documentation polish.

### Install

1. Download `CodexIsland-0.0.3-macOS-unsigned.zip`.
2. Unzip it and move `Codex Island.app` to `Applications`.
3. Launch the app manually.

### Release Notes

- This build is unsigned and not notarized because this environment does not have Developer ID signing, notarization credentials, or Sparkle release keys configured.
- macOS may show security warnings on first launch. Use the standard Gatekeeper override flow if you trust the build.

## 中文

相对于 `v0.0.2`，`v0.0.3` 把更多本地会话体验切到了真实的本地 `codex app-server` 链路上。本地聊天现在可以新开 thread、恢复旧 thread、更稳定地展示 plan 收尾后的后续选项，并让历史记录与 pending interaction 尽量跟随真实本地 thread，而不是退回到容易过期的 transcript-only 状态。本次也继续补强了 notch 命中行为和启动稳定性。

### 新特性

- 本地聊天现在更接近远端工作流：可通过隐藏的本地 `codex app-server` 通道通信，优先使用实时 thread history，并把本地 pending interaction 与真实 thread 状态对齐。
- 本地聊天新增 `/new` 和 `/resume`，同一工作目录下的旧本地线程也能重新打开。
- 本地 app-server 线程现已支持 `/plan`、`/model`、`/permissions` 对应的 slash/config 面板。
- 缩起态 notch 现在会把会话汇总为运行、等待、空闲三类并显示数量，混合本地与远端会话时更容易快速扫一眼状态。

### Bug 修复

- 修复本地 plan 完成后的多种收尾边界情况：无论是 `request_user_input`、`turn_complete`、`task_complete`，还是只输出最终 `<proposed_plan>` 文本的 transcript，后续选项都能更稳定地显示出来。
- 修复本地 app-server 交互的 inline reply，可写状态下 plan / choice 类问题现在可以直接在 UI 里回答。
- 修复缩起态 notch 的 hover 命中区域，并阻止在 notch 内点击 prompt 选项时把 `mouseUp` 误透传给底层全屏终端窗口。
- 通过优先使用可见的本地 app-server threads、支持 synthetic sessions，以及减少历史错绑，修复本地会话恢复和 identity 处理问题。
- 修复调试启动阶段因早期本地 app-server 或 Sparkle 相关写入触发 `SIGPIPE` 导致的启动即退问题。
