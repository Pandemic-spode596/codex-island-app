# Codex Island v0.0.5

[English](#english) | [中文](#中文)

## English

Compared with `v0.0.4`, `v0.0.5` is a focused quality release around chat history fidelity, hook configuration safety, and local session correctness. It also adds a small but useful diagnostics aid in the chat header and tightens the CI environment used for Swift quality checks.

### Highlights

- Chat history now renders image attachments, including content restored from both local transcript parsing and remote thread history mapping.
- Remote thinking history no longer emits empty placeholder items, so recovered conversation timelines are easier to scan.
- Hook installation is safer for existing Codex setups: the app now avoids incorrect `config.toml` rewrites and preserves the file's original newline layout when enabling hooks.
- Local session lifecycle handling is stricter: stop-session teardown short-circuits earlier when appropriate, and notch sessions are limited to hook-bound local processes to avoid false matches.
- When `Remote Debug Logs` is enabled, local and remote chat headers now expose a copyable session or thread identifier to make diagnostics and bug reports easier.
- The GitHub Actions Swift quality workflow now selects a stable Xcode toolchain explicitly and uses an isolated DerivedData path for build and test execution.

### Notes

- Release build version: `0.0.5 (5)`
- From `v0.0.4` to this release, the repository adds `7` commits focused on chat rendering, hook safety, local session state handling, diagnostics UI, and CI stability.

### Install

1. Download `CodexIsland-0.0.5-macOS-unsigned.zip`.
2. Unzip it and move `Codex Island.app` to `Applications`.
3. Launch the app manually.

### Release Notes

- This build is expected to publish as an unsigned, non-notarized zip if Developer ID signing, notarization credentials, or Sparkle release keys are not configured in the current environment.
- macOS may show security warnings on first launch. Use the standard Gatekeeper override flow if you trust the build.

## 中文

相对于 `v0.0.4`，`v0.0.5` 是一次更聚焦的质量发布，重点放在聊天历史还原、hooks 配置写回安全性，以及本机会话识别和清理的正确性上；同时也补了一点诊断辅助能力，并继续收紧 Swift 质量检查的 CI 环境。

### 重点变更

- 聊天历史现在支持图片附件展示，本地 transcript 解析和远端 thread 历史映射恢复出来的图片内容都能直接显示。
- 远端 thinking 历史会过滤空白项，恢复出来的对话时间线更干净。
- hooks 安装对已有 Codex 配置更安全：修复了 `config.toml` 可能被错误重写的问题，并在启用 hooks 时保留原始换行格式。
- 本机会话生命周期处理更严格：停止会话时会更早短路无效 teardown，notch 只会绑定到真正由 hooks 管理的本地进程，减少错误匹配和残留状态。
- 启用 `Remote Debug Logs` 后，本地和远端聊天头部都会显示可复制的 session/thread 标识，便于诊断和提 bug。
- GitHub Actions 里的 Swift 质量工作流现在会显式选择稳定 Xcode，并为构建和测试使用独立的 DerivedData 路径，减少 CI 环境抖动。

### 说明

- 本次发布版本号：`0.0.5 (5)`
- 从 `v0.0.4` 到这次发布，仓库累计新增 `7` 个提交，主要集中在聊天渲染、hooks 配置安全、本机会话状态处理、诊断 UI 和 CI 稳定性上。

### 安装

1. 下载 `CodexIsland-0.0.5-macOS-unsigned.zip`。
2. 解压后把 `Codex Island.app` 移到 `Applications`。
3. 手动启动应用。

### 发布说明

- 如果当前环境没有配置 Developer ID 签名、公证凭据或 Sparkle 发布密钥，本次构建预计会以未签名、未公证的 zip 形式发布。
- macOS 首次启动时可能会给出安全提示；如果你信任这份构建，可以按标准 Gatekeeper 放行流程处理。
